LESSON:    What is a Foreign Key? 


You know that a Primary Key is a column, or combination
of columns, that is used to ensure the uniqueness of a
row in a table.

A Foreign Key is a column, or combination of columns,
in one table that refers back to the Primary Key
of another table.

This relationship between a Primary Key and a Foreign Key
is what is used to connect the data from two tables.





How do you know which books (in the book table) that
a particular publishing company (in the publisher table)
have published?

Look at the Database Diagram.







See that there is a pub_id column that is the Primary
Key of the publisher table.  There is also a pub_id column,
in the book table that is a Foreign Key to the publisher table.




There is a publisher, in the publisher table, that has
a name of "All Techo Books" and it has a pub_id whose
value is 1.

Select all the data in the book table.  Now look at the
pub_id column and you can see that there are six books
that have a pub_id value of 1.   These are the books
that are published by the "All Techo Books" publisher,
since it has a pub_id of 1.

The Foreign Key values in the book table get joined
with matching Primary Key values in the publisher table.



A relational database is designed in such a way to reduce
the redundancy of data stored in the database.  This
necessitates breaking up data into separate tables in
such as way that may not be obvious.

Most of time, questions asked of the database (i.e. queries)
will need to be answered using data from more than one
table.  This is done through a process known as Joining.

Knowing how to Join the data between tables is
absolutely necessary in order to be able to effectively
use the data in most databases!
